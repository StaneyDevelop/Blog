Создание фрагмента
Фрагменты были созданы для того, чтобы ещё сильнее отделить работу с UI и работу с другими модулями приложения. Ими удобно управлять внутри одного Activity. Изменять представление экранов для разных задач, комбинировать их и заменять одни на другие.
Нам необходимо создать один фрагмент для представления экрана, задача которого отображать список. Сначала для всех фрагментов создадим пакет .ui.fragments и внутри него создать класс MainFragment. Унаследуйте класс от android.support.v4.app.Fragment.
Для работы MainFragment необходимо создать его представление. По пути /res/layout создаём файл main_fragment.xml (таким же образом как мы создавали представление для MainActivity).
Далее необходимо внести изменения в класс MainFragment. Первым делом переопределяем метод onCreateView класса Fragment. В нём необходимо установить представление для этого фрагмента и вернуть его в этом методе. Для этого создаём экземпляр класса View. Для его создания можно воспользоваться экземпляром класса LayoutInflater, который идёт входным параметром в методе onCreateView. У него нужно вызвать метод inflate и передать на вход id лейаута и ViewGroup, который мы получили в onCreateView. R.layout.main_fragment - это идентификатор лейаута который мы создали специально для фрагмента. Это всё что необходимо знать для создания View. Более подробно о методе inflate можно узнать из документации https://developer.android.com/reference/android/view/LayoutInflater.html
Теперь фрагмент имеет собственное представление и готов к использованию
